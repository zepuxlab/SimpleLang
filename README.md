# SimpleLang — плагин Figma для локализации фреймов

Плагин дублирует выбранные фреймы (или фреймы с суффиксом `01_EN` / `02_EN`) и подставляет в текстовые слои переводы из списка в формате «локаль + ключ слоя + текст».

---

## Запуск

### Сборка

Из корня проекта:

```bash
npm install
npx tsc
```

В корне появится `code.js` — его использует Figma (в `manifest.json` указан `"main": "code.js"`).

### Установка в Figma

1. В Figma: **Plugins → Development → Import plugin from manifest…**
2. Укажи папку проекта и выбери файл **manifest.json**.
3. Плагин появится в **Plugins → Development → SimpleLang**.

### Запуск плагина

1. Открой файл в Figma.
2. **Plugins → Development → SimpleLang** (или через быстрый поиск по названию).
3. Вставь список переводов в текстовое поле и нажми **«Создать локали»**.

Перед первым запуском можно собрать проект: `npm install && npx tsc`.

## Структура плагина (для разработчика)

### Файлы

| Файл | Назначение |
|------|------------|
| **manifest.json** | Метаданные плагина Figma: `main: code.js`, `ui: ui.html`, `id`, `api`. |
| **code.ts** | Логика в песочнице Figma: парсинг списка переводов, поиск фреймов, клонирование, подстановка текста в слои. Компилируется в `code.js`. |
| **code.js** | Собранный бандл (генерируется из `code.ts`). |
| **ui.html** | UI плагина: текстовое поле для списка переводов, кнопка «Создать локали», поле MCP-токена и кнопки сохранения. |
| **figma.d.ts** | Типы Figma API для TypeScript. |
| **tsconfig.json** | Сборка: target ES2017, `outDir: "."`, в сборку входят `code.ts` и `figma.d.ts`. |

Сборка: только `code.ts` → `code.js`. UI не бандлится, Figma подгружает `ui.html` как есть.

### Сообщения UI ↔ код (postMessage)

**UI → код (pluginMessage):**

- `ready` — UI загрузился; код отвечает `init` и флагом `tokenSaved`.
- `saveToken` — сохранить MCP-токен в `figma.clientStorage` (ключ `simplelang_mcp_token`); тело: `mcpToken`.
- `start` — запуск создания локалей; тело: `translationList` (сырой текст списка переводов).

**Код → UI:**

- `init` — ответ на `ready`: `tokenSaved: boolean`.
- `tokenSaved` / `tokenError` — результат сохранения токена; при ошибке — `message`.
- `status` — текст статуса («Обработка…», «Создание дублей…»).
- `error` / `success` — финальное уведомление с `message`.

Обработчик в коде: `figma.ui.onmessage`. Отправка в UI: `figma.ui.postMessage({ type, message?, ... })`.

### Формат списка переводов

Текст — последовательность блоков локалей. Каждый блок:

1. Строка-заголовок локали: два цифры, подчёркивание, два буквы кода языка — `^\d{2}_[A-Z]{2}$` (например `01_FR`, `02_DE`). Регистр не важен.
2. Строки вида «ключ текст»: ключ — одно из слов `title`, `sub`/`subtitle`, `desc`/`description`; после пробела — значение (до следующей такой строки или до следующего заголовка локали).
3. Многострочные значения: следующие строки без ключа дописываются к последнему ключу через `\n`.

Пример:

```
01_FR
title Télévision intelligente
sub Télécommande
desc Boutons de la télécommande,
joystick et raccourcis.
01_DE
title Smart TV
sub Fernbedienung
```

Парсинг: `parseTranslationList(raw)` → массив `LocaleBlock[]`: каждый элемент — `localeSuffix` (например `01_FR`) и `byLayerName: Map<string, string>` (нормализованные ключи → текст). Нормализация ключей: `normalizeLayerKey` + алиасы `LAYER_ALIASES` (sub → subtitle, desc → desc, title → title).

### Поиск исходных фреймов

Функция `findSourceFrames()`:

1. Если есть выделение на странице — возвращаются выделенные узлы (могут быть несколько).
2. Иначе обход `figma.currentPage.children`: ищется фрейм, в имени которого есть подстрока вида `01_EN` (регулярка `DEFAULT_SOURCE_EN`: `^\d{2}_EN$` в имени). Берётся первый такой фрейм.

Итог: либо явное выделение, либо один фрейм с суффиксом `01_EN` (и при нескольких блоках локалей источник циклически повторяется по индексу `i % sources.length`).

### Алгоритм создания локалей

1. Парсинг: `parseTranslationList(translationList)` → `blocks: LocaleBlock[]`.
2. Источники: `findSourceFrames()` → `sources: SceneNode[]`.
3. Для каждого блока локали (и при нескольких источниках — по одному источнику на блок по кругу):
   - Клонирование исходного фрейма: `source.clone()`.
   - Позиция: тот же `x`, `y` = текущая база (ниже предыдущих дублей с отступом 40px).
   - Имя фрейма: в имени источника подстрока вида `01_EN` заменяется на суффикс локали (например `01_FR`); если замены не было — к имени дописывается суффикс.
   - `detachAllInstances(duplicated)` — рекурсивно отвязываются все инстансы внутри.
   - Сбор текстовых узлов: `collectAllTextNodes(duplicated)` — обход/`findAll` по TEXT, ключ слоя — `normalizeLayerKey(node.name)`; сортировка по `y`, затем по `x`.
   - Заполнение: для каждого текстового узла с непустым `layerName` ищется строка в `block.byLayerName.get(layerName)`; если есть — загружается шрифт узла, в `node.characters` пишется `normalizeTextForDisplay(text)`, при необходимости включается `textAutoResize: "HEIGHT"`.
   - Оставшиеся незаполненные текстовые узлы по порядку сопоставляются с неиспользованными ключами из `FILL_ORDER = ["title", "subtitle", "desc"]` и заполняются так же (fallback по порядку).
4. Новые фреймы добавляются на страницу, viewport прокручивается к созданным узлам; в UI отправляется `success` с количеством блоков.

Шрифты: перед записью в текстовый узел вызывается `loadFontForNode(node)` (поддержка mixed font по диапазонам через `getRangeFontName`).

### Ключи слоёв и нормализация

- В списке переводов ключи: `title`, `sub`/`subtitle`, `desc`/`description`.
- В коде они приводятся к нормализованным: `normalizeLayerKey(name)` + `LAYER_ALIASES`: `sub`/`subtitle` → `"subtitle"`, `desc`/`description` → `"desc"`, `title` → `"title"`. Дополнительно эвристики по подстрокам в имени (subtitle, sub, title, desc).
- Имена текстовых слоёв в Figma сопоставляются с этими же ключами после нормализации; совпадение по ключу даёт, какой текст из блока локали подставить в слой.

---

### Хранение данных

- MCP-токен: `figma.clientStorage` с ключом `MCP_TOKEN_KEY = "simplelang_mcp_token"`. Сохранение/удаление по сообщениям `saveToken` из UI.

---

### Константы и регуляры

- `LOCALE_REGEX = /^\d{2}_[A-Z]{2}$/i` — строка локали.
- `KEY_REGEX = /^(title|sub|subtitle|desc|description)\s+(.*)$/i` — строка «ключ значение».
- `NAME_LOCALE_NN_LL = /\d{2}_[A-Z]{2}/i` — подстрока локали в имени узла.
- `DEFAULT_SOURCE_EN = /^\d{2}_EN$/i` — суффикс фрейма-источника по умолчанию.
- `FILL_ORDER = ["title", "subtitle", "desc"]` — порядок подстановки при fallback по незаполненным узлам.

Использование этой структуры позволяет добавлять новые типы сообщений, форматы ключей или логику поиска фреймов без ломания существующего потока.
